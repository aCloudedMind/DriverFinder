# Driver Finder

Проект для подбора ближайших водителей к заказу с использованием различных алгоритмов.

## Алгоритмы

1. **BruteForce** - Полный перебор всех водителей
2. **KDTree** - Поиск с использованием KD-дерева  
3. **GridPartition** - Разбиение на сетку
4. **PriorityQueue** - Использование очереди с приоритетом

### Выводы

1. **Для малого количества водителей (100)**: 
   - **BruteForce** показывает лучшую производительность
   - Все алгоритмы работают достаточно быстро

2. **Для среднего количества водителей (1000)**:
   - **GridPartition** становится самым эффективным
   - KDTree начинает значительно отставать

3. **Для большого количества водителей (10000)**:
   - **GridPartition** сохраняет лидерство
   - PriorityQueue показывает наихудшие результаты
   - BruteForce удивляет хорошей производительностью для полного перебора

4. **Потребление памяти**:
   - KDTree требует больше всего памяти
   - GridPartition показывает хороший баланс скорости и памяти
